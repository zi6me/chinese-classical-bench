# Idiom Prompting Compression Experiment — Report

## Hypothesis
**典故 = 语义级 RAG 压缩**: 含典故的 prompt 比把典故展开成白话的 prompt 短得多，但下游 LLM 输出质量不输（甚至更好）。

## Setup
- Scenarios: 50 典故 (curated narrative-rich)
- Models tested: claude-opus-4-7, deepseek-3.2, qwen3-coder-next
- Total runs: 3 models × 2 versions × 50 scenarios = 300 calls
- Successful pairs (both versions OK): 146 = 48.7 scenarios × 3 models avg
- Judge: claude-opus-4-7, blind A/B with randomized ordering

## Token Reduction
- literal / idiom **tt-token** ratio (cl100k_base): mean=1.33×, median=1.31×
- literal / idiom **char** ratio: mean=1.31×
- => Average prompt-side token reduction (idiom vs literal): **24.7%**
- literal / idiom **completion-token** ratio: mean=1.09× (model output lengths)

Note: kcli-gw injects a ~3-6k token system prompt before the user prompt, so API-reported
`prompt_tokens` is dominated by that overhead and is reported separately below for transparency.

## Quality (Blind A/B Judge)
Total judged pairs: 140

- **典故 (idiom) wins**: 53/140 = 37.9%
- **Tie**: 18/140 = 12.9%
- **字面 (literal) wins**: 69/140 = 49.3%
- Idiom not-loss rate (win + tie): **50.7%**

### Per-model breakdown

| Model | idiom wins | tie | literal wins | not-loss rate |
| --- | --- | --- | --- | --- |
| claude-opus-4-7 | 16 | 8 | 20 | 54.5% |
| deepseek-3.2 | 12 | 7 | 29 | 39.6% |
| qwen3-coder-next | 25 | 3 | 20 | 58.3% |

## Headline

含典故的 prompt 比白话展开版**省 25% 的 tokens**，且在 48 个场景 × 3 模型的盲评中 **51% 不输 (win + tie)**。

**结论**: token 节省显著 (25%)，质量基本不输 (not-loss 51%)。第二支柱基本立住。

## Limitations

- Sample N=50 scenarios — proof-of-concept, not statistical proof.
- Scenarios generated by Claude Sonnet 4.6, not human-curated — possible LLM-friendly bias.
- Judge is Claude Opus 4.7, which generated the scenarios' framing — self-judging family risk.
- All three target models already understand these 50 common 典故; rare 典故 may behave differently.
- Token counts use tiktoken cl100k_base as a fair proxy; actual provider tokenizers (esp. Qwen/DeepSeek) may compress Chinese differently.
- Completion-token comparison is confounded — literal prompts can also lead to longer answers (more context to elaborate on), inflating completion-token ratio.
- Single trial per (model, scenario, version) — no temperature variance estimate.

## If more time

- Add a 3rd 'idiom + 1-line gloss' variant — strongest setup for production prompts.
- Use human evaluators for a subset to cross-check Opus judge.
- Test on smaller open models (7B / 14B) — compression should break first there.
